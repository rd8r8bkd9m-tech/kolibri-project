## 🎮 Демонстрация функций

### 1. Облачное хранилище

1. Откройте http://localhost:5175
2. Перейдите на вкладку **☁️ Storage**
3. Введите любое имя и пароль (аккаунт создастся автоматически)
4. Нажмите "Войти"

### 2. Загрузка файла

1. В секции **📤 Загрузить файл** нажмите на область или перетащите файл
2. Колибри автоматически сожмет файл перед загрузкой
3. Вы увидите информацию о сжатии:
   - Оригинальный размер
   - Сжатый размер
   - Процент сжатия (% сжато)
   - Использованный алгоритм (DECIMAL10X v1.0)

### 3. Просмотр файлов

В секции **📋 Мои файлы** вы увидите:
- Название файла
- Размер до и после сжатия
- Статистика сжатия (в синей плашке)
- Дату загрузки
- Использованный алгоритм

### 4. Скачивание файла

1. Нажмите кнопку **⬇️** (Download) рядом с файлом
2. Файл автоматически распакуется и скачается оригинальной версией
3. Можете проверить целостность файла (MD5 не изменится)

### 5. Статистика хранилища

Вверху страницы вы видите:
- **Использование хранилища** - прогресс-бар использования (10 ГБ лимит)
- **Статистика Колибри**:
  - Всего оригинального размера
  - Всего сжатого размера
  - Сэкономлено (Мегабайты/Гигабайты)
  - Среднее сжатие (%)

## 📊 Технические детали

### DECIMAL10X Алгоритм сжатия

Kolibri использует комбинированный подход:

```
Исходный файл
    ↓
┌─────────────────────────────────┐
│ 1. Анализ паттернов (DECIMAL10X) │  ← Поиск повторяющихся последовательностей
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 2. Формульное кодирование        │  ← Замена паттернов кодами
└─────────────────────────────────┘
    ↓
┌─────────────────────────────────┐
│ 3. Энтропийное кодирование (zlib)│  ← Дополнительное сжатие
└─────────────────────────────────┘
    ↓
Сжатый файл (с метаданными)
```

### Статистика сжатия

Каждый файл хранит метаданные сжатия:
```json
{
  "originalSize": 1048576,
  "compressedSize": 524288,
  "compressionRatio": 50.0,
  "algorithm": "DECIMAL10X v1.0",
  "blockSize": 4096,
  "patterns": 128,
  "hash": "sha256_digest",
  "timestamp": "2025-11-12T10:00:00Z"
}
```

## 🔐 Безопасность

### Реализованные меры

- ✅ **JWT Authentication** - Токен-базированная аутентификация
- ✅ **CORS** - Кроссдоменная безопасность
- ✅ **SHA-256 Hashing** - Проверка целостности файлов

- ✅ **File Validation** - Проверка размера файлов (макс 100МБ)

### Рекомендации для Production

1. Используйте HTTPS вместо HTTP
2. Установите переменные окружения для JWT_SECRET
3. Включите файловое шифрование на диске
4. Используйте гарантированную архивацию (backup)
5. Мониторьте использование памяти и дискового пространства

## 🐛 Решение проблем

### Ошибка: "Port 3001 already in use"

```bash
# Найти процесс
lsof -i :3001

# Завершить процесс
kill -9 <PID>

# Или использовать другой порт
PORT=3002 node server.js
```

### Ошибка: "Cannot find module"

```bash
# Переустановить зависимости
cd /Users/kolibri/Documents/os/cloud-storage
rm -rf node_modules package-lock.json
npm install
```

### Ошибка: "File upload failed"

- Проверьте размер файла (макс 100 МБ)
- Проверьте место на диске
- Проверьте права доступа на папку `uploads/`

## 📞 FAQ

**Q: Как изменить лимит хранилища?**
A: Отредактируйте `server.js` строка 95: `storage: { used: 0, limit: 10 * 1024 * 1024 * 1024 }`

**Q: Можно ли использовать собственный алгоритм сжатия?**
A: Да, отредактируйте `kompressor.js` и замените `compress()` метод

**Q: Как добавить HTTPS?**
A: Используйте reverse proxy (nginx) или встроенный Node.js HTTPS

**Q: Поддерживается ли распределённое хранилище?**
A: В v2.0.0, пока используется локальное файловое хранилище
