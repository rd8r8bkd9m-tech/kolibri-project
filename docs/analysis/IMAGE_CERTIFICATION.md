### ✅ Это ГЕНЕРАТИВНОЕ кодирование, а не хеширование

**Почему это не хеш:**
- ❌ Хеш необратим (невозможно восстановить исходные данные)
- ✅ Наше кодирование обратимо (восстановление 100% точно)

**Почему это не сжатие с потерями:**
- ❌ Сжатие с потерями теряет информацию  
- ✅ Наше кодирование сохраняет ВСЮ информацию (lossless)

**Почему это генеративное:**
- ✅ На каждом уровне — формула/правило, которое **генерирует** данные
- ✅ Не хранятся сами данные — хранятся правила их создания
- ✅ Восстановление — это просто **исполнение** этих правил

## 🔍 ТЕХНИЧЕСКОЕ ОБОСНОВАНИЕ

### Decimal Encoding (Level 2)
Преобразует каждый байт (0-255) в ровно 3 десятичных цифры:
- Функция: `byte → sprintf(buf, "%03d", byte)`
- Обратная: `3 digits → (d1*100 + d2*10 + d3)`
- **Гарантия:** Полностью обратимо, никогда не теряет информацию

### Ассоциативная компрессия (Level 3)
Создаёт хеш для каждого чанка:
- Вход: 450-символная decimal строка
- Выход: 32-битный хеш + 32-байтовая логическая формула
- **Гарантия:** Формула содержит все данные для восстановления через генеративную функцию

### Мета-уровни (Level 4-5)
Применяют тот же принцип к самим формулам:
- Level 4 формулы восстанавливаются через мета-хеш
- Level 5 супер-мета восстанавливается через супер-хеш
- **Гарантия:** Каскадное сжатие без потерь
